# Contributing

Thank you for your interest in Viontin! Here's how to get started.

## CLA

By submitting any contribution (code, documentation, issues, or pull requests), you automatically agree to the terms in [CLA.md](CLA.md). This ensures the project can remain sustainable in the future.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b feat/my-feature`)
3. **Make your changes**
4. **Test** your changes (`cargo check && cargo test`)
5. **Submit a pull request**

## Code Style

- Follow the conventions in [docs/conventions.md](https://github.com/viontin/docs)
- Run `cargo fmt` before committing
- Run `cargo clippy` and address warnings

## Questions?

Open an issue or start a discussion.

Thank you for helping build Viontin!
